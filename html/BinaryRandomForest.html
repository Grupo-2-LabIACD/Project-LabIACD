Binary Random Forest Classifier¶

Random Forest Classifier Steps:

  • Load Main Dataset
  • Split in Dataset for training and testing
  • Build Bootstrapped Dataset
  • Train Decision Trees on those datasets using diferent groups of features
  • Get Test Data and for each entry on the table make it go through all the trees and take note of the results
  • Agregate all the results from the different trees and choose the best value thorugh majority voting

Model Tuning¶

Now let's see if by changing the parameters we can get a better solution

With sklearn we have some comands that help us with this such as RandomizedSearchCV, but let´s first talk about the parameters we'll be messing with:

  • n_estimators: Represents the amount of Decision Trees in our Forest, we'll try to change the value in increments of 20 to try and find an "optimal" value between [50,400]
  • criterion: criterion in the trees we'll be testing gini vs entropy
  • max_features: the number of features that each tree will have, according to some studies the best values should be around $\sqrt{TotalFeatures}$ or log2(TotalFeatures)
  • min_samples_split: that represents the minimum samples to split a node
  • max_depth: in changing this value we can optimize our model and reduce the risk of overfitting

Stackable Improvements Assumption¶

Note that this is only one proposal of a potential model for this problem where we are going to make some assumptions in order to find if those a are true.
The first of those is that we'll make the superficial and perhaps stupid assumption that if a parameter improves the accuracy when adjusted by himself that, the improvement is going to stack with the improvement that we get from tuning other parameter by themselfs.

By the end we'll see if this assumption holds or not

Data Split 2¶

We'll use the "A scaling law for the validation-set training-set size ratio" paper by Isabelle Guyon findings to try and optimize our data split or an adaptation of the findings to our case.
In the paper it's said that the best split is given by $\sqrt{}$ ln($\frac{N}{hmax}$)
N beeing the number of families of recognizers in an finite H
hmax beeing the largest complexity of those families

So for our random forest we are going to do a small adaptation in the hopes that it improves the accuracy:

  • N is going to be the number of trees in the forest
  • hmax the complexity of the trees.

    We'll use the tree from pyradiomics random forest 1 to make the calculations

Setting hmax¶

The complexity of one tree is determined by bigO(tkd) beeing:

  • t: number training examples
  • k: number of features
  • d: depth of decision tree
